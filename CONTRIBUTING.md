# Contributing

Thanks for your interest in contributing! Here's how to get started.

## Getting Started

1. **Fork** this repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/repo-name.git
   cd repo-name
   ```

## Branching

Create a descriptive branch for your changes:
```bash
git checkout -b feature/your-feature-name
```

Use prefixes like `feature/`, `fix/`, or `docs/` to indicate the type of change.

## Making Changes

- Keep changes focused and minimal — one feature or fix per pull request.
- Write clear, readable code and follow the existing style.

## Running Tests

If the project has tests, run them before submitting:
```bash
# Example — adjust to match the project's test command
npm test
# or
pytest
```

Make sure all tests pass before opening a pull request.

## Submitting a Pull Request

1. Push your branch to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
2. Open a pull request against the `main` branch.
3. Fill in the PR description with a summary of your changes and why you made them.
4. A maintainer will review your PR and may request changes before merging.

## Questions?

Feel free to open an issue if you have questions or run into any problems. We're happy to help!
